🎴 Spaced Repetition SRS (SM-2 Algorithm) Review Flow
SM-2 Engine 
sequenceDiagram
    autonumber
    actor User as 👤 Learner
    participant Screen as 🎴 Flashcard Screen
    participant DB as 🗄️ Expo SQLite
    participant SM2 as 🧮 SRSAlgorithm (SM-2)

    Screen->>DB: 1. SELECT * FROM srs_cards WHERE next_review_at <= ISO8601('now') ORDER BY next_review_at ASC
    DB-->>Screen: Returns Due Flashcards Array
    
    Screen-->>User: 2. Display 3D Flip Card (Front: Blurred Photo + Target Pointer)
    User->>Screen: 3. Taps Card to Flip
    Screen-->>User: Reveal Back: Word, Phonetic, Meaning & Play TTS Audio

    User->>Screen: 4. Selects Grade Rating (0: Again, 1: Hard, 2: Good, 3: Easy)
    Screen->>SM2: 5. calculateNextReview(card, userGrade)
    
    Note over SM2: Execute SM-2 Formula:
New EF' = EF + (0.1 - (5-g)*(0.08 + (5-g)*0.02))
New Interval = Interval * EF
    
    SM2-->>Screen: 6. Return updated card (new next_review_at, EF, interval)
    Screen->>DB: 7. UPDATE srs_cards SET repetitions = ?, interval_days = ?, easiness_factor = ?, next_review_at = ? WHERE id = ?
    DB-->>Screen: Update Success
    Screen-->>User: 8. Advance Progress Bar & Slide to Next Flashcard
📋 SM-2 Algorithm Specs 8 Steps
1
Super-Index Due Query
Leverages idx_srs_due_cards partial index to fetch overdue items instantly.
WHERE status != 'GRADUATED'
2
Front Card Render
Displays blurred context photo with highlighted visual object box.
3
Flip & Audio Synthesis
3D card rotation reveals English word and triggers Expo-Speech TTS.
Speech.speak(word, {lang: 'en'})
5
SM-2 Math Execution
Calculates new Easiness Factor (EF) and review interval based on user recall grade (0-3).
EF' = EF + (0.1 - (5-g)*(0.08 + (5-g)*0.02))
7
Schedule DB Persist
Updates SQLite card record with new ISO8601 target review timestamp.
